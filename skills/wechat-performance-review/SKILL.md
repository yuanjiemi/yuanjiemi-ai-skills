# 公众号数据复盘 Skill

把零散数据变成可比较、可追溯、能指导下一篇内容的复盘记录。

## 强制交互

- 每次在新对话中首次启用时，回复第一行必须原样显示：`已启用「公众号数据复盘 Skill」`。同一任务后续轮次不重复。
- 用户未提供数据时，只问一个问题：`请发送公众号后台截图、Excel/CSV、手动数据，或需要复盘的文章链接。`
- 不要求飞书 App ID、App Secret、Cookie、公众号后台密码或 n8n。
- 不要求用户把世界树 API Key 发到聊天框。

## 选择模式

### 免费手动模式

适用于后台截图、Excel、CSV或用户粘贴的数据。读取能确认的字段；看不清的数字标记“待确认”，不得猜测。

截图至少尽量识别：标题、统计时间、阅读、点赞、分享、爱心/在看。若截图没有标题，请用户补充标题或原文链接。

### 文章链接增强模式

用户要求通过文章链接自动读取互动数据时：

1. 完整阅读 `references/worldtree-api.md`。
2. 运行 `python3 scripts/configure.py --check`。
3. 未配置时运行 `python3 scripts/configure.py --init`，把申请与本机配置指引交给用户；用户暂不配置时切换免费模式。
4. 运行：

   ```bash
   python3 scripts/collect_metrics.py --url "公众号文章链接" --phase "手动复盘"
   ```

   多篇文章可以重复传入 `--url`。完整读取生成的 JSON，再写入复盘库。
5. 只把接口返回的数据当事实；采集失败时说明原因，不补造数字。

API Key 可复用“公众号对标文章采集与分析 Skill”的本机私有配置。Key 禁止打印、回显、写入 Excel、JSON 结果或 GitHub。

## 复盘流程

### 1. 标准化数据

分析前完整阅读 `references/field-schema.json` 和 `references/analysis-rules.md`。

- 统一“阅读量/阅读数”为“阅读量”。
- 统一“转发数”为“分享数”。
- 统一“喜欢/在看/爱心”为“爱心/在看数”，但不得把同一指标重复计入。
- 总互动数 = 点赞数 + 分享数 + 爱心/在看数。
- 互动率 = 总互动数 ÷ 阅读量；阅读量为空或 0 时留空。
- 复盘阶段优先使用用户提供的“第3天、第7天、第30天”；无法确认时写“手动复盘”。

### 2. 判断是否允许比较

- 只有 1 篇：记录绝对数据和内容结构，只给“可能原因”，不得判定高表现或低表现。
- 2 篇：允许做逐项差异，不宣称稳定规律。
- 3 篇及以上：优先使用同一账号、相近统计阶段的历史中位数作为内部基准，再判断高于、接近或低于基准。
- 不把不同账号的阅读量直接横向比较；粉丝规模和分发基础不同。
- 不用单篇相关性推断因果。所有原因写成“可能原因＋验证方式”。

### 3. 连接内容与数据

至少检查：标题承诺、开头方式、内容类型、选题类型、CTA以及统计阶段。

- “表现亮点”写可验证的相对优势。
- “可能原因”最多 3 条，每条说明依据和不确定性。
- “下次优化动作”写 1–3 个可执行动作，避免“继续优化内容”之类空话。
- “可复用经验”提炼结构和方法，不复制原文表达。

### 4. 写入 Excel

- 默认使用 `assets/公众号文章数据复盘模板.xlsx`。
- 用户有现成表格时，优先写入其文件副本，不覆盖原文件。
- 没有文件时，复制模板到当前任务 `outputs/` 再写入。
- 用“原文链接＋复盘阶段”去重；链接缺失时用“标题＋复盘时间”辅助检查并说明局限。
- 新增记录时保留总互动数和互动率公式、日期格式、筛选和冻结窗格。
- 保存为新的 `.xlsx`，例如 `公众号文章数据复盘-20260719.xlsx`。

### 5. 交付

交付：

- 更新后的 Excel 文件。
- 80–150 字复盘结论。
- 3 个表现亮点或问题。
- 1–3 个下一篇优化动作。
- 一个优先验证假设，以及下一次需要观察的数据。

最后询问用户是否要把优化动作交给“根据视频脚本生成公众号文案 Skill”生成下一篇文章。

## 边界

- 默认不定时采集，不自动登录公众号后台。
- 不把互动率等同于转化率；没有新增关注、私域添加、预约或成交数据时，不评价商业转化。
- 世界树是第三方服务，字段、价格和接口可用性由服务方决定。
- 自动写入飞书、Notion或公众号后台属于未来授权增强版，不是免费版前置条件。
