# 视频脚本转公众号文章

把视频语言整理成适合手机阅读的公众号文章，同时保留原作者的事实、观点和个人表达。默认直接交付可复制的成稿，不依赖 n8n、飞书或外部 API。

## 核心约束

- 把当前脚本视为最高优先级事实来源。不得补造人物、数据、经历、工具功能、结果或因果关系。
- 把历史文章和风格样本只用于学习语气、节奏、段落密度和开头方式，禁止复用其中的事实、案例、标题、链接或 CTA。
- 显性保留脚本里的核心对象、场景、痛点和成果，不要用更热门但不相干的主题替换主线。
- 保留作者有辨识度的口语、判断和真实经历；删除口头重复、字幕噪声和无意义衔接。
- 用户指定标题、关键词、案例或修改范围时严格执行。除非用户明确要求，不要擅自改掉已定标题。
- 不输出虚构引语、无法核验的数字或夸张承诺。事实不足时收敛表达，不要用常识补齐具体细节。

## 选择工作模式

根据请求选择一种模式。用户只说“把视频脚本转成公众号文章”时，默认使用“视频承接短文”。

1. **视频承接短文（默认）**：用痛点、场景、故事和悬念引出视频，不提前讲完全部结论；正文中保留视频插入位。
2. **完整图文版**：把脚本的核心过程、方法和结论完整整理成一篇可独立阅读的文章；不需要视频也能读懂。
3. **局部改稿**：以现有草稿为底稿，只修改用户指出的部分；其余标题、事实、结构和 CTA 尽量保持不变。

脚本充足时不要为非关键偏好阻塞任务。只有缺少脚本正文，或模式差异会彻底改变交付物且无法合理推断时，才询问一个简短问题。

## 整理输入

从用户提供的文本或文件中识别：

- 当前脚本或字幕稿
- 用户本次修改要求
- 已有草稿和已定标题
- 可选的风格样本
- 可选的固定 CTA

在内部建立“事实边界”：核心对象、目标读者、真实痛点、具体场景、已有证据、脚本结论、作者原话和禁止添加项。不要向用户展示这份中间分析，除非用户要求。

如果同时出现多个版本，以用户明确标注的当前稿、最新版或待修改稿为准。不要把旧稿中的事实带进新稿。

## 写作流程

### 1. 提炼主线

用一句话确定“谁在什么场景遇到什么问题，作者通过什么观察或方法得出什么结果”。全文围绕这条主线展开。

### 2. 生成标题

默认给出 5 个标题备选并选出 1 个推荐标题。标题必须出现脚本中的核心对象、场景或成果，不得制造正文无法兑现的悬念。

用户已定标题时，直接沿用，不再生成替代标题，除非用户要求优化。

### 3. 组织正文

视频承接短文优先采用：

1. 真实痛点、反常识判断或具体故事开场
2. 补充与目标读者相关的场景
3. 说明这期视频会解决什么问题，但保留关键答案
4. 插入 `[视频号视频插入位]`
5. 衔接固定 CTA

完整图文版优先采用：

1. 具体场景或冲突开场
2. 问题为什么值得处理
3. 脚本中的方法、过程或案例
4. 作者的结论与适用边界
5. 下一步行动或固定 CTA

局部改稿先锁定不可变部分，再按要求修改。若用户只要求“开头更戳痛点”，只强化开头和必要承接句，不要重写整篇文章。

### 4. 适配手机阅读

- 使用短句和自然段，每段通常 1–3 句；不要机械地每句话空一行。
- 小标题只在确实帮助理解时使用，避免把短文切得过碎。
- 删除“大家好”“今天给大家分享”等低信息开场，除非它是作者稳定的个人表达。
- 少用“首先、其次、最后”和空泛总结，优先保留具体名词、动作、场景和结果。
- 默认不使用 Markdown 粗体符号；如果用户需要排版建议，另列少量建议加粗或标红的关键词。

### 5. 处理 CTA

- 用户提供固定 CTA 时，原样保留；只修复明显的错别字或断行，并说明修改。
- CTA 将由其他流程追加时，不生成领取方式、二维码、微信号、预约链接或不存在的福利。
- 未提供 CTA 时，以 `[固定 CTA 由发布流程追加]` 占位，不擅自设计转化承诺。

## 默认交付格式

除非用户指定 JSON、飞书表格字段或其他格式，按以下顺序输出：

1. 标题备选与推荐标题；已有固定标题时只输出固定标题
2. 60–120 字摘要
3. 可直接复制的公众号正文
4. 视频插入位（仅视频承接短文）
5. 用户提供的 CTA 或 CTA 占位

不要输出创作过程、内部事实边界或冗长解释。交付前按 [references/writing-standard.md](references/writing-standard.md) 静默检查；若发现来源冲突或必须由用户确认的事实，再在成稿后用一条简短备注指出。

## 结构化输出

只有用户、自动化流程或调用方明确要求 JSON 时，才使用 [references/writing-standard.md](references/writing-standard.md) 中的 JSON 结构。输出纯 JSON，不加代码块或解释。
